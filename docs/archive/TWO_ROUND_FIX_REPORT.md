## 📋 问题诊断

### 症状对比

| 指标 | 旧版本 (4 月 6 日) | 新版本 (4 月 16 日) |
|------|------------------|-------------------|
| 输出文件大小 | 1.5MB | 76KB (↓95%) |

| 常见误区 | ✅ 有 | ❌ 无 |

### 根本原因

在 **asyncio 重构** 过程中，以下关键方法被**意外删除**：

1. `_generate_keypoint_details()` - 第二轮生成详细内容的核心方法
2. `_build_keypoint_question()` - 构建知识点详情问题的方法
3. `_count_keypoints()` - 统计知识点数量的方法

**旧版本流程:**
```
第一轮：生成主题结构 (subtopics + key_points)
   ↓
第二轮：为每个 key_point 生成详细内容
   - explanation (详细解释 500-800 字)
   - core_concepts (核心概念定义 + 示例)
   - common_misunderstandings (常见误区)
   - practical_application (实际应用)
   - code_example (完整代码示例)
   - self_check_questions (自测题)
   - further_reading (延伸阅读)
```

**新版本流程 (修复前):**
```
第一轮：生成主题结构 (subtopics + key_points)
   ↓
完成 ❌ (缺少第二轮)
```

## 🔨 修复内容

### 修改文件
`workflow_orchestrator.py`

### 新增方法

1. **`_build_keypoint_question()`** (第 735-789 行)
   - 构建第二轮提问的 Prompt
   - 包含详细的 JSON 输出格式要求
   - 针对每个知识点生成深度内容

2. **`_generate_keypoint_details_async()`** (第 791-841 行)
   - 异步版本的第二轮生成方法
   - 遍历所有子主题和知识点
   - 为每个知识点调用 LLM 生成详细内容
   - 将结果添加到 `detailed_keypoints` 字段

3. **`_count_keypoints()`** (第 843-848 行)
   - 统计生成的知识点详情数量
   - 用于进度显示和日志记录

### 修改方法

1. **`_async_execute_task()`** (第 628-675 行)
   - 添加第二轮生成调用
   - 返回包含 `knowledge` 字段的完整结果
   - 更新日志输出格式

2. **`_async_execute_layer()`** (第 569-626 行)
   - 直接使用返回的 `knowledge` (包含详细内容)
   - 降级处理：如果没有 `knowledge` 字段则解析 `content`
   - 在事件总线中添加 `keypoints_count` 字段

## 📝 技术细节

### 并发控制
- 层内并发：3 个任务同时执行
- 知识点详情：顺序生成（避免 API 限流）
- 可调整：修改 `MAX_CONCURRENT_TASKS` 参数

### 缓存机制
- 启用缓存：`enable_cache=True`
- 缓存 TTL：4 小时
- 重复运行可节省 50-70% 的 API 调用

### 错误处理
- 单个知识点失败不影响其他知识点
- 错误信息记录在 `detailed_keypoints[].error`
- 支持断点续传（通过层合并文件）
