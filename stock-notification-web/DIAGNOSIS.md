# 前端链接问题诊断指南

**更新时间：** 2026-03-16 23:50

---

## 🔍 问题现象

- ✅ 测试页面 (`/test-links`) 链接可以正常跳转
- ❌ 主页面和其他页面点击链接不跳转

---

## 🧪 诊断步骤

### 步骤 1：清除浏览器缓存

**最重要！** 浏览器可能缓存了旧版本的页面。

**Chrome/Edge:**
```
1. 按 Ctrl + Shift + Delete (Windows)
   或 Cmd + Shift + Delete (Mac)
2. 选择"缓存的图像和文件"
3. 点击"清除数据"
4. 按 Ctrl + F5 强制刷新页面
```

**Firefox:**
```
1. 按 Ctrl + Shift + Delete
2. 选择"缓存"
3. 点击"立即清除"
4. 按 Ctrl + F5 强制刷新
```

**Safari:**
```
1. 按 Cmd + Option + E 清空缓存
2. 按 Cmd + Shift + R 强制刷新
```

---

### 步骤 2：检查浏览器控制台

```
1. 按 F12 打开开发者工具
2. 切换到 Console (控制台) 标签
3. 访问首页 http://39.97.249.78:5000/
4. 查看是否有错误信息
5. 点击页面上的链接，查看控制台输出
```

**正常情况应该看到：**
```
✅ 页面加载完成
🔗 点击链接：/dashboard target: null
```

**如果有错误：**
- 红色错误表示 JavaScript 错误
- 黄色警告可能影响功能
- 截图错误信息以便排查

---

### 步骤 3：检查 Network 标签

```
1. F12 打开开发者工具
2. 切换到 Network (网络) 标签
3. 点击页面上的链接
4. 查看是否有请求发出
```

**正常情况：**
- 应该看到对新页面的 GET 请求
- 状态码应该是 200 或 304

**异常情况：**
- 没有任何请求：JavaScript 阻止了跳转
- 状态码不是 200：服务器问题

---

### 步骤 4：测试 JavaScript

访问测试页面：
```
http://39.97.249.78:5000/test-links
```

点击"测试 JavaScript"按钮：
- ✅ 显示"JavaScript 工作正常"：浏览器 JS 正常
- ❌ 没有反应：浏览器 JS 被禁用或有错误

---

### 步骤 5：检查浏览器设置

**确保 JavaScript 已启用：**

Chrome/Edge:
```
1. 设置 → 隐私和安全 → 网站设置
2. JavaScript → 允许（推荐）
```

Firefox:
```
1. about:config
2. 搜索 javascript.enabled
3. 确保为 true
```

---

## 🔧 常见解决方案

### 方案 1：强制刷新（最常用）

```
Windows: Ctrl + F5
Mac: Cmd + Shift + R
```

### 方案 2：清除缓存后重启浏览器

```
1. 清除浏览器缓存（见步骤 1）
2. 完全关闭浏览器
3. 重新打开浏览器
4. 访问 http://39.97.249.78:5000/
```

### 方案 3：使用无痕/隐私模式

```
1. Ctrl + Shift + N (Chrome/Edge)
   或 Ctrl + Shift + P (Firefox)
2. 访问 http://39.97.249.78:5000/
3. 测试链接是否正常
```

如果无痕模式正常，说明是缓存或扩展程序问题。

### 方案 4：禁用浏览器扩展

某些扩展程序可能干扰页面功能：
```
1. 禁用所有扩展
2. 刷新页面测试
3. 逐个启用扩展找出问题
```

### 方案 5：更换浏览器

尝试不同的浏览器：
- Chrome
- Firefox
- Edge
- Safari
